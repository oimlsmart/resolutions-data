# minutes/ — Edoxen Minutes shape (one per meeting × language)

Generated by `scripts/parse_minutes.rb` from the GLM-OCR JSON cache at
`reference-docs/.ocr/raw/`. Each file is a single Edoxen `Minutes`
document (see the gem's `Edoxen::Minutes` class and
`edoxen-model/models/minutes.lutaml`).

## Naming

`ciml-{N}-{lang}.yaml` — meeting number N (26–60 covered today),
language code `eng` or `fra`. The cover-page heading is the source of
truth for both.

## Structure

Each file carries:

- `identifier[]` — `[{prefix: "CIML", number: "N"}]`
- `urn` — `urn:oiml:ciml:minutes:ciml-N-{lang}`
- `language_code` + `script` — `eng`/`fra` and `Latn`
- `source_doc` — the original PDF URL when recorded in the OCR cache;
  empty otherwise (provenance is implicit via the SHA-256 cache key
  described in the repo root CLAUDE.md).
- `sections[]` — numbered agenda items with title + Markdown narrative.

The parser is permissive — the source OCR is messy. Section detection
is a heuristic: lines that look like `## {digits} Title...` become
section headings, the paragraphs that follow become the narrative.

## Coverage

16 (meeting × language) tuples today, covering CIML meetings 26–42
and 60. Some meetings have EN + FR; others have one language only
because that's all the OCR cache contains. CIML 23, 29, 30, 31 are
present in the cache but their cover-page ordinal detection failed
(French ordinal parsing edge cases); they will be picked up by future
parser improvements.

## Generating

```bash
ruby scripts/parse_minutes.rb
ruby scripts/link_minutes_to_meetings.rb
```

The first writes everything under `minutes/`. The second updates
`meetings/*.yaml` to reference the minutes URNs and creates skeleton
meeting YAMLs for ordinals that have minutes but no manifest entry
(CIML 26–38). Both are idempotent.

## Validating

Once the edoxen gem ships `validate-meetings` (PR #14 — Minutes model
lands in 0.6.1), the canonical validation surface is:

```bash
edoxen validate-meetings minutes/ciml-38-eng.yaml
```

Until then, the structure is conformant with `Edoxen::Minutes`; field
invariants are documented above.
