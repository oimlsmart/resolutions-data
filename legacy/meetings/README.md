# meetings/ — Edoxen Meeting shape

Generated by `scripts/generate_meetings.rb` from `scripts/manifest.yaml`.
Each file is a single Edoxen `Meeting` document in the canonical
shape (see https://github.com/edoxen/edoxen-model/models/meeting.lutaml
and the gem's `schema/meeting.yaml`).

## Naming

- `ciml-{N}.yaml`     — CIML meeting number N (39–60)
- `conference-{S}.yaml` — OIML Conference session number S (12–17)

## Identification

- `identifier[]` — `{prefix, number}`; CIML meetings carry
  `[{prefix: "CIML", number: "N"}]`; Conferences carry
  `[{prefix: "OIML Conference", number: "S"}]`.
- `urn` — `urn:oiml:ciml:meeting:ciml-N` or
  `urn:oiml:conference:meeting:conference-S`.
- `resolution_refs[]` — cross-link to the matching
  `resolutions/ciml-N-resolutions.yaml` (added when the resolutions
  YAMLs gain their `meeting_urn` field).

## Generating

```bash
ruby scripts/generate_meetings.rb
```

Re-running overwrites the YAMLs. The generator is idempotent.

## Validating

The canonical schema lives at the gem's `schema/meeting.yaml`
(mirrored in `schemas/edoxen-meeting.yaml`). Local validation requires
the next edoxen gem release (0.6.x) which ships
`validate-meetings`. Until then:

```bash
# Once the next edoxen gem is released (with validate-meetings):
bundle install --path=vendor/bundle
edoxen validate-meetings meetings/ciml-56.yaml
```

## Mapping from manifest entries

| manifest field        | edoxen field                     |
|-----------------------|----------------------------------|
| `kind`                | `type` (`plenary`/`conference_session`) |
| `meeting` / `session` | `ordinal` and `identifier[].number` |
| `year`                | `year`                            |
| `date_start` / `date_end` | `date_range.{start,end}`       |
| `city`, `venue`       | `general_area` (also `localizations[].general_area`) |
| `country_code`        | `country_code`                    |
| `venue == "Online meeting"` | `virtual: true`             |
| `lang`                | per-language `localizations[]` + `source_urls[]` |
| `title`               | per-language `localizations[].title` |
| `url`                 | `source_urls[].ref`               |

Committee label is hardcoded per `kind` (CIML → "International
Committee of Legal Metrology (CIML)", Conference → "International
Conference of Legal Metrology").
